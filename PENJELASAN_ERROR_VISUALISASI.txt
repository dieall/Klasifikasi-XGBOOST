## ERROR YANG TERJADI:
```
ValueError: 'labels' must be of length 'x', not 2
```

## PENYEBAB:
Error ini terjadi karena **mismatch antara jumlah data dan jumlah label**:
- Code mencoba membuat 2 label: ['Tidak Layak', 'Layak']
- Tapi data Anda mungkin hanya punya 1 kelas saja

Contoh:
- Jika semua data "Layak" (value = 1), maka label_counts hanya punya 1 elemen
- Tapi code mencoba assign 2 label → ERROR!

## KEMUNGKINAN PENYEBAB DATA ANDA HANYA 1 KELAS:

### 1. Threshold Terlalu Longgar/Ketat
Jika threshold yang Anda set terlalu longgar:
```python
thresholds = {
    'Transmission': (0, 100),    # Terlalu longgar → semua PASSED
    'Tinx': (0, 100),            # Terlalu longgar → semua PASSED
    ...
}
```
→ Semua produk jadi "Layak" (1)

Atau jika terlalu ketat:
```python
thresholds = {
    'Transmission': (99, 100),   # Terlalu ketat → semua FAILED
    'Tinx': (18.8, 18.9),        # Terlalu ketat → semua FAILED
    ...
}
```
→ Semua produk jadi "Tidak Layak" (0)

### 2. Data Produksi Sangat Konsisten
Jika proses produksi Tinstab MT-620 sangat stabil:
- Semua parameter selalu dalam range yang baik
- Tidak ada produk cacat
→ Semua label jadi "Layak"

## CARA MENGATASI:

### Opsi 1: Sesuaikan Threshold (RECOMMENDED)
```python
# Cek statistik data dulu
print(df[numeric_columns].describe())
```

## LANGKAH TROUBLESHOOTING:

1. **Cek distribusi label setelah labeling:**
```python
print(df['Label_Kualitas'].value_counts())
```

2. **Jika hasilnya:**
   - `1    1499` → Semua Layak (sesuaikan threshold lebih ketat)
   - `0    1499` → Semua Tidak Layak (sesuaikan threshold lebih longgar)
   - `1    750, 0    749` → BAGUS! Data seimbang

3. **Analisis parameter yang sering gagal:**
```python
# Cek parameter mana yang sering out of range
for param, (min_val, max_val) in thresholds.items():
    failed = ((df[param] < min_val) | (df[param] > max_val)).sum()
    print(f"{param}: {failed} sampel di luar range")
```
